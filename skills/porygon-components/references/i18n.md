# i18n — `PorygonI18nProvider`

```ts
import {
  PorygonI18nProvider,
  PORYGON_LOCALE_ES, PORYGON_LOCALE_EN, PORYGON_LOCALE_PT_BR,
  PORYGON_LOCALES, // { 'es-ES', 'en-EN', 'pt-BR' }
} from '@envisiongroup/porygon/react-components/i18n';
import type {
  PorygonLocaleText, PorygonLocalePresetCode, // 'es-ES' | 'en-EN' | 'pt-BR'
  EFWTableLocaleText,
} from '@envisiongroup/porygon/react-components/i18n';
```

**Cuándo usar:** traducir los textos internos de la librería (validación de campos, botones CRUD de tablas, overlays, labels de filtros, etc.). Como consumidor normal usas el **provider** o la prop `localeText` de `EFWForm`/`EFWTable`; **no** necesitas los hooks de locale (son de uso interno).

---

## 1. Provider

La firma real tiene **solo dos props**. No existe prop `locale` ni `preset`: el "locale" es el objeto que pasas a `localeText`.

```ts
interface PorygonI18nProviderProps {
  children: ReactNode;
  localeText?: PorygonLocaleText; // parcial; merge profundo con default y providers padre
}
```

Envuelve una vez cerca de la raíz (dentro del `FluentProvider`). Todos los `EFWForm`, `EFWTable`, campos, botones y drawers heredan los textos.

```tsx
<PorygonI18nProvider localeText={PORYGON_LOCALE_EN}>
  <App />
</PorygonI18nProvider>
```

`PorygonLocaleText` es totalmente parcial:

```ts
interface PorygonLocaleText {
  form?: Partial<EFWFormLocaleText>;
  table?: Partial<EFWTableLocaleText>;
  fields?: DeepPartial<EFWFieldsLocaleText>;
  buttons?: DeepPartial<EFWButtonsLocaleText>;
}
```

---

## 2. Presets (ES / EN / PT_BR)

```ts
PORYGON_LOCALE_ES   // default (español); === DEFAULT_* (fallback sin provider)
PORYGON_LOCALE_EN   // inglés
PORYGON_LOCALE_PT_BR// portugués (BR)
PORYGON_LOCALES = { 'es-ES': ..., 'en-EN': ..., 'pt-BR': ... }
```

Pasa el preset directo a `localeText`:

```tsx
<PorygonI18nProvider localeText={PORYGON_LOCALE_PT_BR}>...</PorygonI18nProvider>
```

---

## 3. Schema de textos (resumen por sección)

- **`form`** (`EFWFormLocaleText`): `requiredFieldError`, `requiredFieldsNotification`, `unsupportedFieldType(type)`.
- **`fields`** (`EFWFieldsLocaleText`): agrupado por componente con `labels` y a veces `validationMessages`:
  - `input` (placeholder/emptyState + minLength, maxLength, emailInvalid, chileanRutInvalid, formatInvalid, ...)
  - `numberInput` (+ invalidNumber, minValue, maxValue)
  - `textArea` (+ maxLength), `switch` (checkedText/uncheckedText), `tagPicker`, `datePicker` (+ `calendarStrings`), `attachments`.
  - Varias claves son funciones: `minLength(n)`, `maxValue(value, formatted)`, `maxFileSizeExceededMessage(size, fmt)`.
- **`buttons`** (`EFWButtonsLocaleText`): `button.labels` (loadingText, defaultErrorMessage, ...), `groupButton.labels` (invalidButtonText), `drawer.labels`.
- **`table`** (`EFWTableLocaleText`, ~50 claves): CRUD add/update/delete (labels, drawer titles, action/loading/success/error texts), filtros (placeholders por tipo, labels), overlays (empty/loading/error + retry), command bar (`commandBarSelectedCountLabel(count)`), sort labels. Algunas son `ReactNode` (`*DrawerDescription`) o funciones (`deleteConfirmMultiple(count)`).

Tipos extendidos por componente:

- `EFWForm` → `EFWFormExtendedLocaleText` (form + `fields?` + `buttons?`).
- `EFWTable` → `EFWTableExtendedLocaleText` (table + `form?` + `fields?` + `buttons?`).

---

## 4. Prioridad de override

De menor a mayor (gana el de la derecha):

```text
default (ES)  →  PorygonI18nProvider (externo, anidable)  →  prop localeText del componente
```

Providers anidados se combinan (cada hijo mergea sobre su padre). Lo no especificado cae al default ES.

```tsx
<PorygonI18nProvider localeText={PORYGON_LOCALE_EN}>     {/* base EN */}
  <PorygonI18nProvider localeText={partialFormOverride}> {/* override puntual */}
    <EFWForm fields={fields} />
    <EFWTable fields={tableFields} defaultItems={[]} localeText={partialTableOverride} /> {/* máxima prioridad */}
  </PorygonI18nProvider>
</PorygonI18nProvider>
```

---

## 5. Override parcial (mantener defaults)

El merge es **profundo** (hasta `labels`/`validationMessages`): solo sobreescribes las claves que pasas.

```ts
const partialFormOverride: PorygonLocaleText = {
  form: { requiredFieldError: 'Mandatory field' },
  fields: { input: { validationMessages: { emailInvalid: 'Please enter a valid email address' } } },
};

const partialTableOverride: Partial<EFWTableLocaleText> = {
  addButtonLabel: 'New astronaut',
  overlayEmptyTitle: 'The crew list is empty',
};
```

> Las claves función (ej. `minLength`) se sobreescriben enteras, no se combinan.

---

## 6. Cambio dinámico de locale

Memoiza el `localeText` para evitar re-renders del subárbol:

```tsx
const [code, setCode] = useState<PorygonLocalePresetCode>('es-ES');
const localeText = useMemo(() => PORYGON_LOCALES[code], [code]);

<PorygonI18nProvider localeText={localeText}>
  {/* selector que llama setCode('en-EN' | 'pt-BR' | 'es-ES') */}
  <EFWForm fields={fields} />
</PorygonI18nProvider>
```

---

## 7. Hooks de locale (uso interno)

Resuelven el merge final (default → provider → arg opcional). Los componentes los consumen; como consumidor normal **no** los necesitas:

- `usePorygonLocaleText()` — contexto crudo.
- `useFormLocaleText(localeText?)`, `useTableLocaleText(localeText?)` — aceptan override por argumento.
- `useFieldsLocaleText()`, `useButtonsLocaleText()` — sin argumento.

Úsalos solo si construyes UI propia que deba reutilizar los textos de Porygon.

---

## 8. Gotchas

- No hay prop `locale`/`preset`: el provider solo acepta `localeText`.
- `PORYGON_LOCALE_ES === defaults`; sin provider todo cae a ES.
- `EFWForm`/`EFWTable` envuelven su contenido con un provider **local** cuando detectan `fields`/`buttons`/`form` en su `localeText` extendido — por eso los drawers CRUD heredan traducciones sin provider externo.
- En cambio dinámico, memoiza el `localeText` (`useMemo`).
- Respeta firmas `ReactNode`/función al sobreescribir claves de tabla.

---

## 9. Ejemplos

### A) Provider global con preset

```tsx
<FluentProvider theme={webLightTheme}>
  <PorygonI18nProvider localeText={PORYGON_LOCALE_EN}>
    <EFWForm fields={formFields} />
    <EFWTable fields={tableFields} defaultItems={data} />
  </PorygonI18nProvider>
</FluentProvider>
```

### B) Override parcial sobre un preset

```tsx
const partialFormOverride: PorygonLocaleText = {
  form: { requiredFieldError: 'Mandatory field' },
  fields: { input: { validationMessages: { emailInvalid: 'Please enter a valid email address' } } },
};
const partialTableOverride: Partial<EFWTableLocaleText> = { addButtonLabel: 'New astronaut' };

<PorygonI18nProvider localeText={PORYGON_LOCALE_EN}>
  <PorygonI18nProvider localeText={partialFormOverride}>
    <EFWTable fields={fields} defaultItems={[]} localeText={partialTableOverride} />
  </PorygonI18nProvider>
</PorygonI18nProvider>
```
